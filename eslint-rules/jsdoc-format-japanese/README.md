# jsdoc-format-japanese

日本語 JSDoc コメントのフォーマット規約を強制する ESLint ルール。

## ルールの詳細

日本語ドキュメントの可読性と一貫性を保つため、JSDoc コメントのフォーマットを統一する。

### ❌ エラー (句点がない)

```typescript
/**
 * ユーザーを取得する
 */
export const getUser = async (id: string) => {
  // 文末に「。」がない → エラー
};
```

### ✅ OK (句点がある)

```typescript
/**
 * ユーザーを取得する。
 */
export const getUser = async (id: string) => {
  // 文末に「。」がある
};
```

## フォーマットルール

### 1. 文末は「。」で終わる

日本語の説明文は必ず句点「。」で終わる。

```typescript
/**
 * データベースからユーザー情報を取得する。
 */
```

### 2. 重複句読点の禁止

文末の記号と句点が重複してはいけない。

#### ❌ エラー

```typescript
/**
 * 以下の処理を行う:。
 */
```

#### ✅ OK

```typescript
/**
 * 以下の処理を行う。
 */
```

### 3. 箇条書きの前は空行

箇条書き (bullet list) の前には空行が必要。

#### ❌ エラー

```typescript
/**
 * 以下の処理を実行する。
 * - ユーザー取得
 * - 権限チェック
 */
```

#### ✅ OK

```typescript
/**
 * 以下の処理を実行する。
 *
 * - ユーザー取得
 * - 権限チェック
 */
```

## 自動修正

このルールは自動修正をサポートしている。

### 修正例1: 句点の追加

```typescript
// Before (自動修正前)
/**
 * ユーザーを削除する
 */

// After (自動修正後)
/**
 * ユーザーを削除する。
 */
```

### 修正例2: 重複句読点の削除

```typescript
// Before (自動修正前)
/**
 * 参照:。
 */

// After (自動修正後)
/**
 * 参照。
 */
```

### 修正例3: 箇条書き前の空行追加

```typescript
// Before (自動修正前)
/**
 * 処理内容。
 * - 検証
 * - 保存
 */

// After (自動修正後)
/**
 * 処理内容。
 *
 * - 検証
 * - 保存
 */
```

## 例外パターン

以下の場合は句点不要:

- 英語のドキュメント (日本語文字を含まない)
- `@param`, `@returns` などのタグ部分
- URL や参照リンクで終わる行

```typescript
/**
 * Fetch user data from API.
 * 参照: https://example.com/api/users
 *
 * @param id - User ID
 * @returns User object
 */
```

## メリット

1. **可読性**: 統一されたフォーマットで読みやすい
2. **プロフェッショナル**: 文書としての品質向上
3. **一貫性**: プロジェクト全体でスタイルが統一される
4. **自動化**: 手動での調整が不要
