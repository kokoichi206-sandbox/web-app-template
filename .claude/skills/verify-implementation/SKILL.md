# 実装の動作確認

実装したものの動作確認を行います。Playwright MCP を使用してスクリーンショットを撮影し、変更が正しく反映されているか確認します。

## 手順

1. **出力ディレクトリの作成**
   - `output/` 配下に、確認対象を示すディレクトリを作成してください
   - 例: `output/remove-slide-feature/`, `output/fix-login-button/`
   - ディレクトリ名は変更内容を端的に表すものにしてください

2. **開発サーバーの起動確認**
   - `pnpm dev` が起動しているか確認してください
   - 起動していない場合はバックグラウンドで起動してください

3. **Playwright MCP を使用してスクリーンショットを撮影**
   - 確認が必要なページにアクセスしてください
   - 必要に応じてスクロールやクリックなどの操作を行ってください
   - スクリーンショットを撮影し、作成したディレクトリに保存してください
   - ファイル名は `01-page-name.png`, `02-after-click.png` のように連番をつけてください

4. **確認結果のレポート**
   - スクリーンショットの内容を確認し、期待通りの表示になっているか報告してください
   - 問題があれば指摘してください

## 引数

- `$ARGUMENTS`: 確認対象の説明 (例: "スライド生成機能の削除", "ログインボタンの修正")

## 注意事項

- Playwright MCP は headless モードで動作します
- スクリーンショットは `output/` 配下に保存されます (git 管理外)
- 開発サーバーが起動していない場合は自動的に起動を試みてください
